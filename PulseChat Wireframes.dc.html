16 · UX Review & Build Recommendations
How to read this section
Every mobile/desktop frame above shows one "happy path" state. This section lists the states each screen also needs (loading, empty, error, confirmation) plus screen-specific UX notes — so engineering doesn't have to guess what happens on a slow network, a first-run account, or a failed request. Frame IDs (A1, F2, N5…) refer back to the wireframes above.
Global patterns to build once, reuse everywhere
Loading skeletons
Grey block skeletons for: explore grid/feed (B1–B3), profile grid (D1/D2/L5), chat list rows (F1), chat message history on open (F2/F5), notifications (C2/L4), search results as user types (C1/L3). Never show a blank white screen >150ms.
Empty states
Needed for: no posts yet on own profile (D1), no messages yet in a new chat (F2), no search results (C1), no notifications (C2), no saved posts (K2), no friend requests (H1), no blocked users (N2). Each needs a short line + one relevant action, not just an icon.
Error / offline states
Failed post/message send (retry chip on the bubble itself, not a modal), failed image upload in composer (G2/K4/N20), no-connection banner app-wide, failed login (A1) vs. account-not-found vs. wrong-password — these should be 3 distinct messages, not one generic error.
Confirmations
Destructive or hard-to-reverse actions need an explicit confirm sheet like N1/N14: delete post, delete message (N6), leave/delete group (N4), unfriend, block (N1), log out on shared devices, revoke session (E2).
Toasts & inline confirmations
Lightweight, non-blocking feedback for: link copied (H1 invite link), saved/unsaved a post, blocked a user, friend request sent, settings saved. Don't reuse the destructive confirm-sheet pattern for these — it's too heavy.
Permission prompts
Camera/mic access before Go Live or a call (N17/N19), photo-library access before attachment picker (N10), push-notification opt-in — should appear as an in-app priming screen before the native OS prompt, explaining why.
Per-screen notes
A1 Login
Add inline error states for wrong password vs unknown username vs suspended account (link to N-series moderation actions). Disable "Log in" button until both fields have input, not just on submit.
A2 Sign up
Username-taken check should be inline/async (not only on submit). Password strength meter needs a "too weak" red state, not just green.
A4 New device
Add an expiry/timeout state ("This link expired, request a new one") — email confirmation links usually expire in ~15–60 min.
B1/B2 Explore
First-run empty state before the user has any friends/interests — trending/hashtags only, no "people you may know" yet. Pull-to-refresh spinner. Infinite-scroll loading footer at bottom of feed.
C1 Search
Needs a "Recent searches" state before typing, and a distinct "No results for '…'" empty state after typing — currently only the populated state is drawn.
C2 Notifications
Mark-all-read action and a filter (All / Mentions / Requests) will matter once volume grows — worth a v2 pass, not blocking for launch.
D1 Own profile
Empty-grid first-run state ("Share your first post") is missing. Avatar upload needs a crop/preview step, not drawn yet.
D2 Other profile
Add a "Request sent — pending" state (button greyed with a cancel option) distinct from the initial "Add friend" state, and a "You've been blocked" dead-end state.
E1/E2 Settings
Toggle changes should show a brief inline "Saved" pulse rather than a full toast per tap. Password-change flow (E2) needs its own dedicated screen (current + new + confirm), not just an entry row.
F1 Chat list
First-run empty state ("Message a friend to get started"). Swipe actions (mute/delete/pin) should be specified — currently only visible via the 📌/🔕 icons with no interaction shown.
F2/F5 Conversation
Send-failed state on a bubble (red exclamation + "Tap to retry"), and a connecting/reconnecting banner for the realtime socket. Media messages need an upload-progress ring before delivery.
G2 Composer
Add an upload-progress state for the photo/video, a "discard draft?" confirm on Cancel if content was entered, and a caption character-limit indicator.
G3 Post detail
"No comments yet — be the first" empty state, and comment-post pending/failed states for slow connections.
H1 Friends
Zero-state for "no requests, no suggestions yet" on a brand-new account. Confirm/delete on a request should show optimistic removal with an undo toast, not just vanish.
I1/I2 Admin
Reports queue needs pagination/loading, and an audit-trail confirmation after "Review" resolves a report (what action was taken, timestamped, for accountability).
K1/M1 Status poll
Show a brief "voted ✓" confirmation state on the option after tapping, and disable re-voting (or explicitly allow changing vote — decide and reflect it in the tap state).
N1 Block
After confirming, show a brief toast ("sam_writes blocked") and pop back two levels (skip the now-inaccessible profile), not just one.
N3/N4 Groups
Minimum-members validation (need ≥2 others) before "Next" is enabled. Removing yourself as last admin needs a guard/prompt to reassign admin first.
N13 Data export
Needs an "export in progress" state and a "download ready" state (link + expiry), not just the request form shown.
N17/O3 Go live
Camera/mic permission-denied fallback, and a "your stream ended" summary screen (duration, peak viewers, total comments) — currently nothing exists after the stream stops.
N19 Call
Missed-call / declined-call / call-ended-with-duration states, and a poor-connection indicator — only the ringing state is drawn today.
