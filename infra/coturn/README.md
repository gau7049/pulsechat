# coturn on the Oracle Always-Free VM (manual provisioning)

Human-owner steps (Build Instructions §4 — requires SSH access the agent doesn't have).
Estimated time: ~30 minutes once the VM exists.

## 1. VM + network

1. Create an Always-Free VM (Ubuntu 22.04+, Ampere or Micro shape).
2. Point a DNS record at it, e.g. `turn.<your-domain>` (or use the public IP and adjust
   `realm`/`server-name`/`cert` paths in `turnserver.conf`).
3. Open ingress in the VCN security list **and** the VM's firewall:
   - `3478/udp`, `3478/tcp` (STUN/TURN)
   - `5349/tcp` (TURN over TLS)
   - `49152–65535/udp` (relay range)

## 2. Install

```sh
sudo apt update && sudo apt install -y coturn certbot
sudo certbot certonly --standalone -d turn.<your-domain>
sudo mkdir -p /var/log/turnserver && sudo chown turnserver: /var/log/turnserver
```

## 3. Configure

```sh
sudo cp turnserver.conf /etc/turnserver.conf     # this repo's file, hostnames replaced
echo 'TURN_SECRET=<long random string>' | sudo tee /etc/coturn/secret.env
sudo chmod 600 /etc/coturn/secret.env
sudo systemctl edit coturn
```

Drop-in contents (injects the secret without writing it into the config file):

```ini
[Service]
EnvironmentFile=/etc/coturn/secret.env
ExecStart=
ExecStart=/usr/bin/turnserver -c /etc/turnserver.conf --static-auth-secret=${TURN_SECRET}
```

```sh
sudo systemctl enable --now coturn
```

## 4. Wire to the API

Set the **same** secret as `TURN_SHARED_SECRET` and the hostname as `TURN_HOST` in the
API host's environment (see `.env.example`). The API issues per-user, short-lived
credentials via the TURN REST scheme — no static credentials ever reach the client.

## 5. Verify

Use https://webrtc.github.io/samples/src/content/peerconnection/trickle-ice/ with
`turn:turn.<your-domain>:3478` and a credential generated by the API (M5 exposes a
dev endpoint for this) — expect `relay` candidates.
